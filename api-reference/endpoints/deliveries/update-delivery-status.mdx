---
title: "Update Delivery Status"
api: "POST /deliveries/update_status"
---

### üìù Summary

This endpoint allows updating the status of a specific delivery. The status can reflect various stages of the delivery process, such as pending, in transit, completed, or failed. This is typically used when there is a change in the delivery‚Äôs current state, helping to track its progress.

---

### Requests
##### Params

<Expandable>
**Description:** 

|Name|Location|Type|Required|Description|
|---|---|---|---|---|
|`body`|body|object| no |none|
|`delivery_uuid`|body|string| yes |none|
|`status`|body|string| yes |accepted | in_transit | delivered | cancelled|
|`pickup_otp`|body|string| no |required if status is in_transit|
|`dropoff_otp`|body|string| no |required if status is delivered|
|`note`|body|string| no |Reason for cancellation|
</Expandable>

##### Example

<Expandable>
```yaml
delivery_uuid: "{{delivery_uuid}}"
status: in_transit
pickup_otp: "097857"
dropoff_otp: "098798"
note: Reason for cancellation

```
</Expandable>
